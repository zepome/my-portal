# 🚀 Zepome's Portal

モダンなダークテーマの個人用ポータルサイトです。天気情報、カレンダー、Todo管理、スプレッドシート表示、タイマー/ストップウォッチなどの機能を統合した、オールインワンのダッシュボードです。

## ✨ 機能

### 📊 統合ダッシュボード
- **ヘッダー**: Zepome's Portal ロゴ、天気情報(ミニ)、タイマー(ミニ)、時計、Gmail未読数
- **リアルタイム時計**: 現在時刻を常に表示
- **ダークモードUI**: 目に優しい黒背景にネオンカラーのアクセント

### 🌤️ 天気情報
- 滋賀県、大阪府、東京都の天気予報
- 気象庁APIからリアルタイムデータ取得
- ヘッダーにコンパクトに表示

### 📅 カレンダー
- 月次カレンダー表示(全幅・大きく表示)
- Todoの期日を自動的に表示
- **時間情報も含めた予定表示**（例: 「10:00 資料作成」）
- **カレンダーから直接スケジュール追加**: 日付にホバーで「+」ボタン表示
- 各日120px以上の高さで見やすく

### ✅ Todoリスト
- **全幅の横スクロール表示**: ヘッダー直下に配置
- **タスクの作成・編集・削除**: 完全なCRUD操作
- **グループ分け機能**: 仕事、個人、買い物など
- **日程・時間設定**:
  - 日程: 白いカレンダーアイコンをクリック→インラインカレンダーで選択
  - 時間: プルダウンで00:00〜23:00まで選択可能
- **カレンダーへの自動反映**: 設定した日程と時間でカレンダーに表示
- **編集機能**: 各Todoに鉛筆アイコン、クリックで編集モーダル
- **完了/未完了のチェック**: チェックボックスで管理
- **データ自動保存**: LocalStorageで永続化

### 📊 スプレッドシート
- 複数のGoogleスプレッドシートを表示
- タブで簡単に切り替え
- **URLを貼り付けるだけで自動ID抽出**
- カレンダーの下に大きく表示(600px高さ)

### ⏱️ タイマー/ストップウォッチ
- ヘッダーにコンパクトに表示
- タイマーとストップウォッチの切り替え
- **0.1秒単位の表示**（00:00:00.0形式）
- 一時停止・リセット機能
- タイマー表示クリックで設定モーダル

### 📧 Gmail連携
- 未読メール数の表示（要Google API設定）

## 🛠️ セットアップ

### 1. ファイル構成

```
portal/
├── index.html      # メインHTML
├── style.css       # スタイルシート
├── script.js       # JavaScriptロジック
├── config.js       # 設定ファイル
└── README.md       # このファイル
```

### 2. GitHub Pagesで公開

#### 手順1: GitHubリポジトリ作成
1. [GitHub](https://github.com)にログイン
2. 右上の「+」→「New repository」
3. リポジトリ名を入力（例: `my-portal`）
4. 「Public」を選択
5. 「Create repository」をクリック

#### 手順2: ファイルをアップロード
1. 作成したリポジトリを開く
2. 「uploading an existing file」をクリック
3. すべてのファイル（index.html, style.css, script.js, config.js, README.md）をドラッグ&ドロップ
4. 「Commit changes」をクリック

#### 手順3: GitHub Pagesを有効化
1. リポジトリの「Settings」タブを開く
2. 左メニューの「Pages」をクリック
3. 「Source」で「Deploy from a branch」を選択
4. 「Branch」で「main」を選択し、「/ (root)」を選択
5. 「Save」をクリック

#### 手順4: URLにアクセス
数分後、以下のURLでアクセス可能になります:
```
https://[あなたのユーザー名].github.io/[リポジトリ名]/
```

### 3. Google API設定（オプション）

Gmail未読数表示を使用する場合:

1. [Google Cloud Console](https://console.cloud.google.com/)でプロジェクト作成
2. Gmail APIを有効化
3. 認証情報でAPIキーとOAuthクライアントIDを作成
4. `config.js`の以下を編集:

```javascript
API_KEY: 'あなたのAPIキー',
CLIENT_ID: 'あなたのクライアントID.apps.googleusercontent.com',
```

**注意**: 現在はローカルストレージでTodoを管理しているため、Google API設定なしでも基本機能は動作します。

## 🎨 カスタマイズ

### 色の変更
`style.css`の`:root`セクションで色をカスタマイズできます:

```css
:root {
    --neon-blue: #00d4ff;      /* メインアクセント */
    --neon-purple: #b84fff;    /* セカンダリ */
    --neon-pink: #ff006e;      /* 強調 */
    --neon-green: #00ff88;     /* 成功 */
}
```

### 天気の地域追加
`config.js`で地域を追加できます:

```javascript
WEATHER_API: {
    shiga: 'https://www.jma.go.jp/bosai/forecast/data/forecast/250000.json',
    osaka: 'https://www.jma.go.jp/bosai/forecast/data/forecast/270000.json',
    tokyo: 'https://www.jma.go.jp/bosai/forecast/data/forecast/130000.json',
    // 他の地域を追加...
}
```

地域コードは[気象庁API](https://www.jma.go.jp/bosai/common/const/area.json)で確認できます。

## 🎯 使い方

### Todoの追加
1. 「Todo リスト」の「追加」ボタンをクリック
2. タスク名を入力
3. グループを選択（または新規作成）
4. 白いカレンダーアイコンをクリックして日程を選択
5. 時間をプルダウンから選択
6. メモを入力（任意）
7. 「保存」をクリック

### カレンダーからスケジュール追加
1. カレンダーの日付にマウスを乗せる
2. 右上に表示される「+」ボタンをクリック
3. Todoモーダルが開き、日程が自動入力される
4. タスク名と時間を入力
5. 「保存」をクリック

### Todoの編集
1. Todoリストの鉛筆アイコンをクリック
2. 内容を変更
3. 「更新」ボタンをクリック

### スプレッドシートの追加
1. 「スプレッドシート」の「追加」ボタンをクリック
2. シート名を入力
3. GoogleスプレッドシートのURLを貼り付け
4. 「保存」をクリック

### タイマーの使用
1. ヘッダーのタイマー表示をクリック
2. 時間を設定
3. 「設定」ボタンをクリック
4. 「▶」ボタンでスタート

## 🔒 プライバシー

- すべてのTodoデータはブラウザのLocalStorageに保存されます
- 外部サーバーにデータは送信されません
- 天気情報は気象庁の公開APIを使用
- スプレッドシートはGoogleのiframe埋め込みを使用

## 🐛 トラブルシューティング

### 天気情報が表示されない
- ブラウザのコンソールでエラーを確認
- CORS制限の可能性（ローカルで開いている場合）
- GitHub Pagesで公開すると解決します

### スプレッドシートが表示されない
- スプレッドシートの共有設定を「リンクを知っている全員」に変更
- URLが正しいか確認

### Todoが消える
- LocalStorageは同一ドメインでのみ保持されます
- ブラウザのキャッシュをクリアするとデータが消えます
- ブラウザのシークレットモードではデータが保存されません

### カレンダーにTodoが表示されない
- Todoに日程が設定されているか確認
- ページを再読み込み（Ctrl+Shift+R）

### 更新が反映されない
- ブラウザのキャッシュをクリア（Ctrl+Shift+R）
- GitHubのActionsタブでデプロイ状況を確認
